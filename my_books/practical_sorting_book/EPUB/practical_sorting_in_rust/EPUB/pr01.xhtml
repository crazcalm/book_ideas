Preface

This book is about sorting. I acknowledge that "sorting" means many things to many people. As a Software Engineer, I would define sorting as the act of placing a collection of items in order. For example, if I take the list of [4, 3, 5, 2, 1], the sorted order will be [1, 2, 3, 4, 5].

If you agree with this definition, I agree with you. If you disagree with this definition, I also agree with you. My issue with this definition is that it is both correct and misleading. It implies that "sorting" can only result in one definitive order, but what is "order"?

The American Heritage dictionary defines order as a condition of logical or comprehensible arrangement among the separate elements of a group.

This means that as long as the final arrangment of elements is logical or comprehensible, they are ordered. With that in mind, lists like [1, 2, 3, 4, 5] and [5, 4, 3, 2, 1] are sorted because both ascending and descending order of numbers are logical arrangments. The same thing can be said for the list [1, 3, 5, 2, 4]. It's sorted because the numbers are placed in ascending order where the odd numbers are placed in front of the even numbers, which is a logical arrangement.

When it comes to the word "order", the important part is the logical arrangement. For example, the list [3, 1, 2, 4, 5] could be unordered, or it could be sorted in ascending order where 3's are given priority and placed at the front of the list. The distinguishing feature between unordered and ordered is resproducibility. If you generate the list [3, 1, 2, 4, 5] on your first attempt, but re-running the code tends to generate different lists, then it is safe to say that this is not a logical arrangement and that your list is not sorted.

Throughout this book we will be examining sorting with respect to ordering. We will perform this examination in Rust. That said, the learnings and general takeaways discovered should be easily transferable to the language of your choice. WIth respect to Rust, we will learn how to leverage sorting to achieve a cerain logical arrangement or order, we will learn how to implement sorting for custom structs, and we will learn how to exend our sorting capabilities at runtime to allow for dynamic sorting.
